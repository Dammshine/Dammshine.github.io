Stack and Queue

Stack

Stack is the abstracted data structure that the element can only inserted or remove only from one end. While this makes stack less flexible, it also makes stack’s implementation more flexible. “LIFO”

One implementation is using array. By using an array that will expand when the size if full, we can acheive O(1) for both push and pop.

Another implementation is to use linked list, where linked list will be more flexible for the memory usage. As the memory used is directly corrlate with number of element, while array can have potentially wasted space.

The most common usage for stack is implementing recursion, which can be used to reduce the coefficient greatly.

Queue

Queue is a data structure, that it’s element can only inserted at one end and retrive at the another. “FIFO”

One implementation would be using linked list, a wrapper struct store both start and end of the list.

Another implementation would be using circular array, which index may be calculated by ultilize modular property.

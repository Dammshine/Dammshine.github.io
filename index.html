7.0 Introduction to Array

Array is an ordered collection of elements, and each elements have an numeric index. In JS, the elements is untyped which means two element in array can be different type. And JS handle dynamic memory allocation. JS array can be sparse which means it does not garunteed a continguous block of memory.

JS array is a specialize form of JS object, it similar to views array is object with property happens to integer, however it normally gives a better performance.

JS array inherits from Array.prototype which defines a rich set build-in method.

7.1 Creating Array

There are several ways to create array, which include here

Array Literal

The syntax is simply a comma-separted array element within bracket, and one can omit element by just fill nothing between two comma, that creates a sparse array,

Spread Operator

The spread operator of the array works similar as object, …a basically replaced with a’s object.

The spread operator can be used to create a deep copy of an array, and it works for all iterable object (are which for/of loops over).

Array() construcctor

One can invoke Array() in three distinct ways

  • No argument. Array() create an empty array.
  • Single numeric argument. Array(10) create a sparse array with length 10.
  • More than one argument, equivalent as put argument as elements in literall array initilization. Avoid.
Array.of() and Array.from()

Array.of() is same as Array Literal, all argument treat as number.

Array.from() expect a iterable object, which is same as Spread Operator. It can convert array like object to true array. Array.from() can have a second argument, each element copied will pass through that function.

7.2 Reading and Writing Array Elements

The rules is basically identical to how to access Object, using []. All index are property name with integer between 0 and 2^32 - 2. This also allows you to create property that is anything, such as negative number, this is also the reason that outbound access will return undefined because the array don’t have property stored.

7.3 Sparse Arrays

Sparse array is one of the elements in array don’t have contiguous index from 0. It’s got subtle difference element between undefined, and empty although both return undefined. And underlying implementation allows sparse array got a better implementation compare to simply define missing index be undefine. Normally we works with dense array, the code will treat empty element to be undefine in non-sparse array.

7.4 Array length

Every array have a length property and this is what makes array different from regular JavaScript object. For the non-sparse array, length will be one greater than the greatest index. For sparse array, it is also garunteed to be one greater than the greatest index.

In order to maintain this invariant, it have following bahavior

  • the array’s length will modify each time we assigns value to greater index.
  • Each time modify length below the greatest index, the index greater or equal to the length get deleted.

7.5 Adding or Deleting element in array

The folowing method will be handy

  • push() expect one or more elements, push to the end of array and modify the length
  • pop(), expect an element remove a element in the array, and modify the length.
  • unshift(), add a element at begining of array and shift the existing element by 1, increment the length.
  • shift(), return the first element at begining of array, and shift back the existing element by 1, decrement the length.

The few things worth noting,

  • push() does not flatten array, so add … is a superior choice.
  • shift(), if there are more than one argument, multiple elements will be push in begin at the same time.
    One can also delete element by delete, delete don’t affect the length property, it’s similar assigning undefine to the element.

splice() will cover in detail in 7.8 .

7.6 Iterating Arrays

The for/of loop is covered in chapter 5, it’s one of the way to iterating array. Note, it don’t have any special behavior for the sparse array as it simply return undefine.

If one need pair(index, element) (C++ nerd check this out), one can use entries() method

If you want to do operation to each element, (lambda expression!), one can use forEach() which expect a function that have the individual element as it’s argument, or an additional index as second argument.

JS don’t have true Multidimentional Array. But you can approximate using array of array.

6.1 Introduction to Object

An object is a composite value, it aggregate multiple values and allows you to store and retrive.

A property has a name and value, the name can be string, symbol but no properties can have duplicate name.

From OOP point of view, property can be classify as

  • Own property
  • Inherited property

Beside above, each property have three property attributes, (By default, three property is set to be true)

  • The writable attribute specifie whether the value of property can be set.
  • The enumerable attribute specifies whether the property name is retruned in for/in loop.
  • The configurable attribute specifies whether the prpperty can be deleted and whether its attribute can be altered.

6.2 Creating Objects

Object Literal is the easiest way to create object, which is a comma seperated list enclosed with curly braces. Each time object literal execute, it create new and unique object, and if one want compare two identical object both created by this, it will return false.

Second method is create object using new, the new keyword must be followed by a constructor function. JavaScript build-in type included the constructor

Third method Object.create() creates a new object, uses its first argument as the prototype of that object, and second argument describe property of new object,

One example of usage is to use this to create a new object that inherits from other object, and pass the newly created object to the function

6.3 Querying and Setting Properties

Access property

To obtain value of all property, it can use ‘.’ or ‘[]’.

  • To use ‘.’, it must followed by name without space, hyphen,
  • use ‘[]’ it must followed by something that can be convert to string, or a value or a symbol and that string/number/symbol correspond to the property.

The way using ‘.’ is aligh with must strong typed language, and since it uses the identifier which isn’t a type in JS language, it can’t be modified. On the other hand, using ‘[]’ accept anything that can be converted to string, and it can be manipuated. In some case, we won’t able to know the identifier’s name in advance and so one must use ‘[]’ which gives much more flexibility.

Inheritance

JavaScript object have a set of “own property” and a set of “Inherited property “. When search property x from object o, JavaScript will first inspect if x is o‘s own property, if it’s not, it will then repeat this process on o‘s prototype until it either found the property or the prototype is null which meant it’s the fundamental prototype.

Assign same property name will always create or reassign the own property, the property that exist in inherited property will be hidden if own property have same name. When assignment, inherited property only examined to determine if assignment is allowed. For example, if name is read-only property for prototype, then the assignment is not allowed. However, if it’s allowed, it will create a new same name own property and never modify the prototype’s property. The only exception is when property is a accessor property. Detail in 6.10.

Property Access Error

Attemp to access object will return undefined for the property doesnt exist. However, access property within undefined or null will cause error. Use ?. or && operator to guard this kind of error.

–Attemp to set object on null or undefined also cause TypeError, set other value cause problem when some property is read-only and cannot set. –

6.4 Deleting Property

Delete remove property, it evaluated true in following case

  1. Delete property successful.
  2. Delete did nothing. (Property do not exist, property isnt own property)
  3. Delete is meaningless.

It fail at following case

  1. Property is non-configurable. Fail to delete, and evaluated as fail.

6.5 Testing Property

Here is three ways to examine if the property is within object, using either string or symbol.

  1. in operator return true is property exist either in own property or inherited property.
  2. hasOwnProperty() only test own object
  3. propertyIsEnumerable() only test own object and check if attribute is enumerable will be explore in chapter 14

And another way to examine is to check if property is undefined.

6.6 Enumerating Properties

There are few ways to examine enumerable property. The for/in loop runs the body of loop once for all enumerable property (own or inherited), assigning name to the variable

As an alternative, one can first get an array of property names, then using for/of loop to access the array, the following method all gives only own property

  • Object.keys() return an array of names of the enumerable own properties that are string(does not include symbol).
  • Object.getOwnPropertyName() works as Object.keys() but include non-enumerable own properties as well (still not include symbol).
  • Object.getOwnPropertySymbol(), return name of symbol, enumerable and non-enumerable.
  • Object.ownKey(), return every own property.

The rules of order are given below

  • Property name are non-negative will interpret as integer, put first and order by size.
  • Property name are string, ordered by the sequence added to the object.
  • Property name are symbol, ordered by the sequence added to the object.

for/in() loop will not tightly specific this order, the implementation will typically follow the above then repeat same process on the chained prototype.

6.7 Extending Object

A common operation in JS is needing to copy the properties of one object to another, and JavaScript have build-in Object.assign(),

Object.assign() takes two or more arguments and will copy-set property from second argument to the first argument, it will override property is the name is the same.

Union-Find Disjoint Sets

2022-05-16

Union-find Disjoint set

Union-find Disjoint sets (UFDS) is an abstracted data structure that want to simulate a collection of elements belongs to a collection of sets, where we need to support the following two operations

  1. Determine if two element belongs to same set,
  2. Combine two disjoint set.
    Such data structure can be helpful in situation such as finding connected component of a diagram, krusal’s algorithm.

Linked list Implementation

We would want to support the above operation with decent efficiency. The main paradigm for supporting such data structure is to choose one unique element to represent the set.

Then, a naive approach would be using a linked list, where each node contains a parent pointer points back to the head, which is used to represend the set. It gives constant time theta(1) to determine if two element belongs to same set by looking up the parent pointer’s address of this two element.

To combine two disjoint sets, denote set S1 and set S2. The operation would be choice one set from set S1, set S2 to be merged and let opposite one represent the new set. For example, we let set S2 to be merged, set S1 become new set, then we would change all nodes’s parent pointer in set S2 to set S1’s head, and then combine the two linked list. As showned by the diagram. This operation will takes theta(length of S1 or S2) depend on which set we choice to merge.

Combine - size resolution

As one can observe above, while determine same set will have constant running time, the combine operations takes operation linearly to the length of linked list. In order to maximise efficiency, we would always want to merge the shorter length linked list to the longer ones. We can store an additional attribute, denote size so that we have constant time for look up the size of set. With length, we reduce time complexity to theta(MIN(S1, S2)).

By using this additional size, it can boost time efficieny for our data structue. Consider the following scenrio, suppose there are initially n elements each contained in n disjoint sets, then n of combine operation will takes theta(nlog(n)) time. To prove this time complexity is true, we consider one arbitary sets denote S1 and operation spend on this set. For each combination step between S1 and S2(a random set), the resulting is either

  1. S1’s pointer being updated, then the result new set’s size is strictly greater than length of S1, this takes theta(S1),
  2. S1’s pointer not being updated, then takes no operation.
    What this means, is that for x’s size <= n, each object’s pointer being updated for at most O(log(n)) times, and the total time complexity for above operations will be O(nlog(n)).

Rooted Tree Implementation

If we develop further on the idea of choosing one unique element to represent the set. A more efficient way would be using a general tree-like structure, with the root of tree used to representing the set. And general tree-like structure will be implemented using an array.

Then, to determine if two set are equal, we will need to find the root of two set by recursively visit it’s parent and finally we would compare the root.

If we want to combine two set, choice the root to represent the new set and simply points the merged ones to the new root. As shown by the diagram.

Combine - rank resolution

Similar idea can be also apply to rooted tree implementation, when combine two sets, we would want the root of the tree with fewer node points to the root with more nodes. To do so, we can store an additional attribute rank for each of nodes, that indicate upper bound of the height of the node.

Combine - path compression

Idea of path compression is when we inspect the path denote P, from some node to it’s root, we makes the node directly to the root. By doing so, it reduce the path from node to the parent. The following imgae shows this operation would do.

To achieve the above idea, we change our findSet function slightly, as it recurse, it makes a path between node and it’s root, and as it rewinds back, it will update node’s parent directly to the root.

As a result of above improvement, for m is number of disjoint-set operation, n is number of elements, the worst-case running time will be O(m * a(n)) where a(n) is a very slowly growing function. Further detail can be found in CLRS Chapter 21.4.

About version control

Version control is a system that record changes of a file or a set of file, so that one can revert back, compare difference over time, who last modify which file with what… with very little workload.

Local Version Control System

One of popular VCS tools was a system called RCS(Revision Control System).nRCS works by keeping patch(change) set in a special format, so it can then recreate any file looked like at any point by adding up all patches.

Centralized Version Control System

The next major development would be Centralized Version Control System(CSCSs), which is a single server that contains all the versioned file, and multiple clients that check out files from the central server.

  • Pro
    1. Everyone have a shared project. Easy to work as a team compare with LVCS.
    2. Admin have fine-grained control.
  • Con (This also occurs for LVCS)
    1. Safety. Centralized means risk of losing everything.
    2. Single point of failure of Centralized server represent whole failure. Such as if the server updates, everyone can’t do anything.

Distributed Version Control System

Distributed Version Control System(DVCS) solves above issue by letting every user contains a fully mirror of repository that include all previous histroy. So that if above safety issue happens, if there is one client have the repository then everything will be fine. And secondly, it offers more flexibity, by deal preety with with working simutaneously on the same project.

Git Basic

Git have the following important property

Snapshot not difference

The most other VCS uses snapshots, they maintain the information of changes of file, over the time. Git’s doesn’t think of the data in this way, it basically took a “snapshot” of every file at the time, and whole version of histroy will more like a stream of snapshots.

By the above property, it allows most operation in git only require local files.

Git Has Integrity

To avoid corruption of file, lose of information, git uses SHA-1 hash. This hash also prevent changes of contents of file without Git know about it.

Another characteristic is git generally only add data to its database. As it uses snapshot. By this, one can experiment many thing with git without worry about screw things up.

The Tree States

Git has three main states that your files can reside in

  1. Committed means file stored in local database (Git directory)
  2. Modified means you have changed the file but have not commited and not staged to your database.
  3. Staged means you have marked a modified file in its current version to go to next commited shot.

Git have three main section

  1. Git directory is where git store metadata and object database for the project.
  2. Working directory is single checkout of version of the project. Like a snapshot. (When using clone)
  3. Staging area, is a file that track information of your next commit, which is from working directory to Git directory.

Tries (Radix tree)

Tries

One property observe for the tree is that for different permutation of record, the shape of BST would differ with extreme case that BST become a linked list. This is called object space decomposition, because it’s decompositiuon of key range is driven by the object.

An alternative to object space decomposition is to predefine the spliting position. So that the tree’s insertion patteren will be independent from the value, order of inserting item. This technique is called key space decomposition.

  1. This won’t garunteed the tree to be balanced.
  2. The depth of tree will be fixed. (If know the range of data)
  3. The value stored in the leave.

A data structure based on key space decomposition is called tries. Huffman coding tree would an example of a binary tries.

Alphabet tries

The tries we refer normally refer to the alphabet tries. It split using character in string. One application would be dictionary, once constructed it used to determine if a given work is in dictionary.

Cost analysis

  • O(n) space if linked list, O(k*m) if using array
  • O(m) insertion and search
    for n total size of text
    for m length of key string
    for d size of underling alphabet

Sorting

General Property of Sorting

Sorting problem is defined

Stable / unstable

In particular, many sorted data we met will be in form as record, which each record contains a key and satellite data. In practice, we wish to permute the satelliete data as well.

We define stable, as if before sorting a[i] == a[j], then

  • Stable means after sorting, a[i]’s new position is before a[j].
  • Vice versa

Adaptive

The bahaviour/performace of algorithm affected by data values (distribution of sorted data).

In place

If the algorithm uses constant amount of memory. (And we wish this amount to be relatively small compare with input size)
External sorting is class of algorithm that handle, large amount of data that store in external device, when data does not fit into main memory of computing device. Result the data must reside slower external memory. The in-place property will be very valueable.

O(n^2) sort

Bubble sort

Selection sort

Insertion sort

Shell sort

Summary of Performance

O(nlog(n)) sort

Comparison sort

From all above, these algorithm shares an intersting property, the sorted order they determined is basd only on comparison between input elements. We call above algorithm comparison sort.

Decision Treee

In comparison sort, we use only comparison between two element to gain info about ordering. We may construct a model to analysis the general performance of comparison sort. A decision tree is a full binary tree that represents the comparison between elements that are performed by a particular sorting algorithm.

Since each sorting algorithm may result n! different outcome, so the leave node in the full binary tree would be at least n!. The lower-bound then would be height of tree, which will be nlog(n).

Quicksort and heapsort are asmptopically optimal, as both’s worst case match with this lower bound.

non-comparative sorting algorithm

There is algorithm that do not base on comparison, thus not bounded by the lower bound of decision tree.

Counting sort

Counting sort is a stable sort, and require extra memory use. It’s stable property is important for other reason, for radix sort may work using counting sort.

Algorithm describe

This algorithm have time complexity of O(n + k), consider k <= n for most used cases, which will be O(n). It seems out-performed to nlog(n) sort, however consider it usage cases, it only perfrom good with k is small. In most sorting scenoio, we use int, which have 2^32 possibility, but the size of input often do not meet this number. Hence it’s direct uses case is limited, tho it can perform well when combined with Radix sort.

Radix sort

Radix’s sort algorithm is very simple, it applys stable Counting-sort the index from end to front, it requires stable sort property as if not stable, the sequence determined in higher index might not be in sorted order. Such as 13, 14, one can’t tell the sequence by only inspect index ‘1’, and thus must be stable and ensure lower-index’s sequence determined before higher index.

The time complexity would be O(m*n), where n is number of index and m is the possibility in index. Radix sort perform better than comparison sort.

There is few disadvantage of Radix sort

  1. Although it tooks less pass for n element, but each pass may be longer (such as int, either takes 4 sort on the byte or 1 sort in 32 bits, not ideal as one comparison)
  2. Quicksort ultilize hardware cashes more effectively than radix.
  3. The stable sort algorithm always take extra memory, which many O(n*log(n)) uses in place memory.
  4. Conclusion would be, which sorting algorithm we use depend on the chracteristic of data we sort, implementation of the machine and the memory.

BST

Property of BST

  1. The right sub-tree of the root always have value greater than the root, left sub-tree always smaller than the root.
  2. Perfectly-balanced tree, abs(node on left subtree - node on right subtree) <= 1.
  3. Height-balanced tree, abs(height on left tree - height on right tree) <= 1.

Time complexity for search, delete, insert is typically O(height)
One property observe for the tree is that for different permutation of record, the shape of BST would differ with extreme case that BST become a linked list. This is called object space decomposition, because it’s decompositiuon of key range is driven by the object.

Insertion and search is very primititive. Let’s look at deletion.

For tree deletion, we consider following four cases

  1. If node deleted is leave. Then break the link, we are done.
  2. If node deleted have one subtree, replace link it’s children.
  3. If node deleted have two subtree, replace root by the smallest node on it’s right subtree.

Balancing Search Tree

For the general BST, there are two improvement one can make.

  • Insertion at Root
  • Tree partition

For insertion at root, essentially after insertion, we make several rotation to make sure that this new node become the new root. It got two benifit, one is normally newly inserted data likely being inspected, second is it can make the tree more balance, but not garunteed.

For tree partiton, it require each node store an additional attribute contain the node count from each subtree.

After my experiment, conclusion would be this two operation itself isn’t helping very much.
The partition will only ensure that it balance its root, which is not ideal because what really affetcing performance would be the height. Which perfect balance not help. A way to balance would be balance at every node using partition function.

Tree traversal

There are four traversal

  1. Pre-order
  2. In-order
  3. Post-order
  4. Level-order (iterative or using queue)
    A few variant would be travel right before left.

Stack and Queue

Stack

Stack is the abstracted data structure that the element can only inserted or remove only from one end. While this makes stack less flexible, it also makes stack’s implementation more flexible. “LIFO”

One implementation is using array. By using an array that will expand when the size if full, we can acheive O(1) for both push and pop.

Another implementation is to use linked list, where linked list will be more flexible for the memory usage. As the memory used is directly corrlate with number of element, while array can have potentially wasted space.

The most common usage for stack is implementing recursion, which can be used to reduce the coefficient greatly.

Queue

Queue is a data structure, that it’s element can only inserted at one end and retrive at the another. “FIFO”

One implementation would be using linked list, a wrapper struct store both start and end of the list.

Another implementation would be using circular array, which index may be calculated by ultilize modular property.

Makefile

Compiler : Gcc

Compilers are programs that convert program source code to executable form, “executable” might be machine code which are those machine can read (C, C++), or bytecode which is low-levl code can be processed by software interpreter (java).

In C, the compiler have following stages

  • Pre-processing
    1. In this stage, the compiler will do things such as, macro substitution, testing for condition, directives and file inclusion.
  • Compilation
    1. In this stage, the compiler will translate source code to the language that machine can read.
  • Linking
    1. In this stage, the compiler will link multiple programs as one program.

Gcc compiler, for compilering

  1. -c : This command compile the gram give the object file as output. Which can be used in linking stage.
  2. -E : This command gives pro-processed file.
  3. gcc -o name a.o b.o : Linkes a.o, b.o into executable program.
  4. gcc a.c b.c d.c: This command compile a.c, b.c , d.c as a executable program, there must include a main in .c files.

Gcc compiler, for advanced

  1. -Werror: Will compile the source and show warning.
  2. -Wall : Will check error, but treat all warning as error.

Makefile Syntax

Compilation process is complex for large systems, and the make command assist by allowing

  • Programmers to document depencies in code (Off topic : Topological sort).
  • Minimal re-compilation
    • Based on dependencies, and also the time of latest change.

A dependency specify

and the target is rebuild if the source changed after last compile.

Few more things to note:

  • A target is always on a new line,
    • Followed by a ‘:’
    • Followed by files that it depend on
    • Make can also compile the target speicify in makefile
  • The action(gcc part) is always on a new line
    • and must indented by a TAB

If no argument is given, the make command only build the first target in makefile, to ensure all file getting compiled, one should specify a target thet contains all files we want to build as the first target.

Beside target and action, makefile also have variable

  • CC refers to the compiler (gcc in this case)
  • CFLAGS contains compiler directives/options (-Wall and -Werror)
  • LDFLAGS is a list of link (or “load”) directive (Not seen so far)

ADT

Abstract data Type

A data type is

  • A set of values
  • A collection of operations on thoes value

An abstract data type is

  • An approach to implementing data types, that seperate users and implementer,
    • Implementer will provide interface (how to use) and implementation
    • User only need to know interface.

Many of ADT deals with

  • Collection of items, that often contain a key(or themselve is a key)
    • Particular structure that provide great efficency, but user want to reduce complexity for them to use it more easiy. So we hide the detail.
      • linked list, tree, graph
    • Particular usage, that user only need the result.
      • matrix, set, dictionary, stack, queue

We want to distinguish

  • DT (Non-abstract data type)
    • Internal of da structure is visible. (character, C string)
  • GADT
    • C++ template like data type. Can have mutiple type. (C++ STL, vector)

Performance Analysis

Analysis of Algorithm

Analyzing an algorithm, have come to mean of predicting the resource of algorithm required for size of problem. By analyzing several canidate algorithms, we can identifying the most desirable one.

There is a few terms need to be well defined here

  1. Input size, the best notion for input size depend on the problem, such as for sorting number, the input size is number of integer as the input.
  2. Running time, one should adopt the notion that our model of analysis assume CPU takes constant operation every seconds, thus running time can also be reconigized as the number of primititive operation that CPU supported.
  3. Memory, one correlate of additional memory from input, that the algorithm required.

Order of growth

We shall make one more simplifying abstraction, which is the rate of growth, which is ignore all coefficent and smaller rate of growth term in estimating algorithm’s performance. When we look at input size large enough, it eventually will make only the order of growth of running time relevant.

We now define big O notation, this will give us upper bound,
O(g(n)) = {f(n) : there exist positive constant c and n0 such that f(n) < c * g(n) for all n > n0.}

Suppose two function have same Big-Oh function, then two function have same order of growth. Furthemore, a bigger-order growth function can’t be Big-Oh function for a lower-order growth function. As there do not exist a positive constant n that can component the difference brought by the order-growth difference.
